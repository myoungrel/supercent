# 기술 선택 근거

## 시스템 구조: LLM + RAG

이 프로젝트는 **LLM 단독 사용이 아닌 LLM + RAG 구조**를 채택했다.

| 방식 | 결과 |
|------|------|
| LLM 단독 | "광고가 많으면 유저가 이탈할 수 있습니다" → 학습 데이터 기반 일반론 |
| LLM + RAG | "Hill Climb Racing 유저가 실제로 이런 불만을 남겼습니다 (유사도 54.9%)" → 실제 데이터 근거 |

RAG를 통해 한국 플레이스토어 1~2점 리뷰 1,918건을 벡터DB에 저장하고, 기획안에서 추출한 검색 쿼리로 유사 불만 패턴을 검색한다. LLM은 이 검색 결과를 컨텍스트로 받아 근거 기반 리포트를 생성한다. LLM이 학습 시점 이후 데이터나 한국 특화 반응을 알 수 없는 한계를 RAG로 보완하는 구조다.

## LLM - Claude API (claude-sonnet-4-6)

| 항목 | Claude Sonnet | GPT-4o | Gemini 1.5 Pro |
|------|--------------|--------|----------------|
| 한국어 이해 | ★★★★★ | ★★★★☆ | ★★★★☆ |
| JSON 출력 안정성 | ★★★★★ | ★★★★☆ | ★★★☆☆ |
| 긴 컨텍스트 처리 | ★★★★★ | ★★★★☆ | ★★★★★ |
| 비용(입력 $/MTok) | $3 | $5 | $3.5 |

**선택 이유**
- 한국어 리뷰 분석 및 기획 문서 이해 품질이 높음
- 구조화된 JSON 출력 안정성이 높아 파싱 오류 최소화
- 기획안 특성 추출 → 리스크 리포트 생성까지 긴 컨텍스트 처리에 적합
- GPT-4o 대비 동급 성능에 비용 저렴

**전처리용 - claude-haiku-4-5**
- 3,636건 대량 처리에 비용 효율적 (Sonnet 대비 약 20배 저렴)
- 단순 분류 작업(불만 유형 태깅, 노이즈 필터링)에 충분한 성능

## 임베딩 - Voyage AI voyage-3

| 항목 | Voyage AI voyage-3 | OpenAI text-embedding-3-small | Cohere embed-v3 |
|------|-------------------|-------------------------------|-----------------|
| 무료 제공 | 200M 토큰 | 없음 | 없음 |
| 벡터 차원 | 1024 | 1536 | 1024 |
| 한국어 품질 | ★★★★★ | ★★★★☆ | ★★★★☆ |
| 비용($/MTok) | $0.06 | $0.02 | $0.10 |

**선택 이유**
- 무료 200M 토큰으로 프로토타입 임베딩 비용 0원
- OpenAI 대비 한국어 의미 검색 품질 우수
- 1024차원으로 저장 공간 효율적이면서 검색 정확도 유지

**RAG 검색 임계값 설정 근거 (0.5)**
기획안(자연어 문서)과 리뷰 데이터(구어체 단문)는 텍스트 형식이 달라 코사인 유사도가 구조적으로 낮게 측정되는 특성이 있다. 실험적으로 0.5~0.7 구간이 "의미적으로 관련 있는 불만 패턴"에 해당함을 확인하여 임계값을 0.5로 설정했다. 0.7 이상으로 설정 시 관련 데이터가 있음에도 검색 결과가 반환되지 않는 문제가 발생했다.

## 벡터DB - Supabase pgvector

| 항목 | Supabase pgvector | Pinecone | Weaviate |
|------|------------------|----------|----------|
| 무료 플랜 | ✅ (500MB) | ✅ (제한적) | ✅ (제한적) |
| 별도 인프라 | 불필요 | 필요 | 필요 |
| 일반 쿼리 병행 | ✅ PostgreSQL | ❌ | ❌ |
| Next.js 연동 | 공식 SDK | REST API | REST API |
| 관리 복잡도 | 낮음 | 중간 | 높음 |

**선택 이유**
- PostgreSQL 기반으로 벡터 검색과 일반 쿼리를 단일 DB에서 처리
- Pinecone 등 별도 벡터DB 없이 인프라 단순화
- 무료 플랜으로 프로토타입 운영 가능
- `match_complaints` RPC 함수로 코사인 유사도 기반 RAG 검색 구현 용이
