## 1. 문제 정의

### 해결하려는 문제

하이퍼캐주얼 게임은 짧은 주기로 빠르게 기획·제작·테스트가 이루어지기 때문에, 기획 단계에서 유사 게임의 유저 불만 패턴을 충분히 검토하고 반영하기 어렵다.

플레이스토어 리뷰에는 광고 피로, 조작 불편, 반복성, 난이도 급상승, 보상 부족 등 실제 유저가 이탈하는 이유가 담겨 있지만, 이를 사람이 직접 수집하고 정리하기에는 시간과 비용이 많이 든다.

결과적으로 시장에서 반복적으로 나타난 불만 요소가 기획안에 반영되지 못한 채 출시 후 동일한 문제가 반복된다.

### 현재 업무 방식의 한계

| 단계 | 문제 |
|------|------|
| 유사 게임 분석 | 담당자가 직접 플레이하며 감각에 의존 |
| 리뷰 수집 | 수동으로 읽고 정리 — 비효율적 |
| 기획 검토 | 정량적 근거 없이 경험 기반 판단 |
| 출시 후 | 저평점 → 원인 파악 → 수정 반복 |

### 대상 사용자

- 하이퍼캐주얼 게임 기획자
- 퍼블리싱 담당자 / 경쟁 게임 리서치 담당자
- 초기 프로토타입 검토를 진행하는 PM

## 2. 해결 방법

**RiskPatch** — 플레이스토어 저평점 리뷰를 RAG로 구조화하여, 게임 기획 단계에서 유사 불만 패턴을 자동 탐지하는 AI 리서치 도구.

기획안을 입력하면 실제 유저 리뷰 데이터를 기반으로 잠재적 리스크를 위험도별로 정리하고, 구체적인 기획 보완 방향을 제시한다.

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 기획안 구조화 | 장르 / 조작방식 / 광고구조 / 보상구조 / 난이도흐름 자동 추출 |
| Multi-query RAG 검색 | 7가지 불만 유형별 쿼리를 병렬 생성 후 벡터 DB 검색 |
| 리뷰 노이즈 제거 | 욕설·단순감정·짧은 리뷰 제거 후 핵심 불만 유형으로 구조화 |
| 리스크 리포트 생성 | 위험도(🔴🟡🟢) + 근거 리뷰 + 기획 보완 제안 |
| 실시간 진행 표시 | SSE 스트리밍으로 각 분석 단계를 실시간 반영 |

## 3. 아키텍처

### 데이터 파이프라인 (사전 구축)

```
플레이스토어 1~2점 리뷰 스크래핑
(20개 게임 / 3,636건)
        ↓
Claude Haiku — 노이즈 제거 + 불만 유형 구조화
(욕설·단순감정 제거 → 1,778건)
        ↓
Voyage AI voyage-3 — 임베딩 (1024차원)
        ↓
Supabase pgvector — 벡터 저장
```

### 사용자 워크플로우

```
기획안 텍스트 입력
        ↓
[Step 1] Claude Haiku
기획안 구조화 + 7가지 불만 유형별 검색 쿼리 생성
        ↓
[Step 2] Voyage AI + Supabase pgvector
7개 쿼리 배치 임베딩 → 병렬 벡터 검색 → 중복 제거 → 상위 10건
        ↓
[Step 3] Claude Sonnet
RAG 결과 기반 리스크 리포트 생성 (위험도 + 근거 + 보완 제안)
        ↓
리스크 리포트 출력 (SSE 스트리밍)
```

### 기술 선택 근거

| 기술 | 선택 이유 |
|------|----------|
| Claude Haiku | 구조화 단계는 단순 JSON 추출 — 빠르고 저렴 |
| Claude Sonnet | 리포트 생성은 복잡한 맥락 판단 필요 — 최신 모델 |
| Voyage AI voyage-3 | 200M 토큰 무료, 한국어 성능 우수, 1024차원 |
| Supabase pgvector | 무료 플랜, 별도 벡터DB 없이 PostgreSQL 확장으로 처리 |
| Next.js App Router | SSE 스트리밍 지원, Vercel 배포 최적화 |

## 4. 기능 소개

### 기획안 입력
- 장르, 조작방식, 광고구조, 보상구조, 난이도흐름을 자유 텍스트로 입력
- 샘플 기획안 제공으로 즉시 테스트 가능
- 입력된 기획안은 분석 목적으로만 사용, 별도 저장 없음

### 분석 단계 실시간 표시
- 기획안 구조화 → 불만 패턴 검색 → 리스크 리포트 생성
- SSE 스트리밍으로 실제 서버 진행 상황을 정확하게 반영

### 유사 불만 패턴 카드
- 실제 플레이스토어 저평점 리뷰에서 추출한 유사 사례 표시
- 불만 유형(광고피로·난이도·보상부족 등) 분포 뱃지
- 유사도 수치 표시

### 리스크 리포트
- 위험도 높음(🔴) / 중간(🟡) / 낮음(🟢)으로 분류
- 각 항목: 불만 유형 + 근거 리뷰 요약 + 기획 보완 제안
- 종합 의견 + 리포트 복사 기능

## 5. 고도화 방향

### 데이터 확장
- 크롤링 게임 수 20개 → 200개 이상
- 정기 크롤링 자동화 (주 1회 신규 리뷰 수집)
- 앱스토어(iOS) 및 영어권 리뷰 추가

### 검색 정확도 개선
- 불만 유형별 가중치 적용 (장르별 차별화)
- Cross-encoder 기반 검색 결과 리랭킹
- LangGraph 자가교정 루프 도입
  - RAG 유사도 낮을 시 LLM이 쿼리 자동 수정 후 재검색
  - 7가지 불만 유형 커버리지 체크 후 누락 항목 보완 (데이터 100개 게임 이상 확보 후 도입 권장)

### 기능 확장
- 분석 히스토리 저장 및 기획안 버전별 리스크 비교
- 팀 공유 기능 (링크로 리포트 공유)
- PDF 리포트 다운로드

### 보안
- Google OAuth / 사내 SSO 인증 (현재: IP 기반 Rate limiting 구현)
- Redis 기반 분산 Rate limiting
- 온프레미스 LLM 옵션 검토 (기획안 외부 전송 최소화)

### 슈퍼센트 특화
- 내부 출시 게임 데이터 연동
- 출시 후 실제 평점과 사전 리스크 예측 정확도 검증
- 리스크 예측 모델 파인튜닝
