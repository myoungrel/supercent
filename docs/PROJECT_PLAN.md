## ① 문제 정의

### 해결하려는 문제

하이퍼캐주얼 게임은 짧은 주기로 빠르게 기획·제작·테스트가 이루어지기 때문에,
기획 단계에서 유사 게임의 유저 불만 패턴을 충분히 검토하고 반영하기 어렵다.

특히 플레이스토어 리뷰에는 광고 피로, 조작 불편, 반복성, 난이도 급상승, 보상 부족 등
실제 유저가 이탈하는 이유가 많이 담겨 있지만, 이를 사람이 직접 수집하고 정리하기에는
시간과 비용이 많이 든다.

그 결과, 이미 시장에서 반복적으로 나타난 불만 요소가 다음 게임 기획안에 충분히 반영되지 못하고,
출시 이후 비슷한 문제가 다시 발생할 수 있다.

### 대상 사용자
- 하이퍼캐주얼 게임 기획자
- 게임 퍼블리싱 담당자
- 경쟁 게임 리서치 담당자
- 초기 프로토타입 검토를 진행하는 PM

### 현재 업무 방식
1. 기획자/퍼블리싱 담당자가 유사 게임을 직접 찾아 플레이하며 시장 반응 파악
2. 플레이스토어/앱스토어 리뷰를 수동으로 읽으며 분석
3. 별점이 낮은 리뷰를 일부 모아서 참고하지만, 대부분 개인 경험/감각에 의존
4. 출시 후 저평점 → 원인 파악 → 수정 → 재테스트 반복

## ② 해결 방법

### 기획하는 AI 제품
**Game Risk Patcher (RiskPatch)** - 게임 기획안 기반 유저 불만 리스크 사전 진단 도구

기획안을 입력하면 플레이스토어 저평점 리뷰 데이터에서 유사한 불만 사례를 검색하고,
예상 유저 불만 요소와 기획 리스크를 사전에 알려주는 AI 리서치 도구.

### 핵심 기능
1. **게임 기획안 구조화**: 입력된 기획안에서 장르, 조작방식, 광고구조, 보상구조, 난이도흐름 추출
2. **저평점 리뷰 기반 RAG 검색**: 벡터DB에서 기획 요소와 관련된 불만 사례 검색
3. **리뷰 노이즈 제거**: 욕설·단순감정·짧은 리뷰 제거 후 핵심 불만 유형으로 정리
4. **기획 리스크 리포트 생성**: 위험도 + 근거 리뷰 + 기획 보완 제안
5. **기획 보완 포인트 제안**: 광고 빈도 조정, 튜토리얼 강화, 후반 난이도 완화 등 실질적 수정 방향

### 사용자 이용 흐름
```
기획안 입력
    ↓
핵심 요소 추출
(장르 / 조작방식 / 광고구조 / 보상구조 / 난이도흐름)
    ↓
저평점 리뷰 RAG 검색
    ↓
불만 패턴 분석
    ↓
리스크 리포트 생성
(위험도별 정렬 + 근거 리뷰 + 기획 보완 제안)
```

## ③ 사용할 AI 도구

| 역할 | 도구 | 이유 |
|------|------|------|
| 리뷰 전처리 (노이즈 제거 + 구조화) | Claude API (claude-haiku-4-5) | 빠르고 저렴, 대량 처리 적합 |

| 임베딩 | OpenAI text-embedding-3-small | 성능/비용 균형 |
| 벡터 저장/검색 | Supabase pgvector | 무료, 설정 간단 |
