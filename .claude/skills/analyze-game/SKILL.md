$ARGUMENTS 내용을 분석해서 리스크 리포트를 생성하세요.

## 분석 단계

1. **기획안 핵심 요소 추출**
   입력된 기획안에서 다음 항목을 파악:
   - 장르 (러너, 퍼즐, 아이들, 클리커 등)
   - 핵심 조작방식 (탭, 스와이프, 기울기 등)
   - 광고구조 (강제광고/보상형/배너, 노출 빈도)
   - 보상구조 (보상 지급 방식, 진행 구조)
   - 난이도흐름 (초반/중반/후반 난이도 곡선)

2. **RAG 검색**
   추출된 핵심 요소로 벡터DB에서 유사 불만 패턴 검색
   (CONTEXT_ENGINEERING.md의 Stage 3 참고)

3. **리포트 출력 형식**
   ```
   ### 게임 기획 리스크 진단 리포트

   **분석된 기획 특성**
   - 장르: ...
   - 광고구조: ...
   - 보상구조: ...
   - 난이도흐름: ...

   **리스크 항목**

   🔴 높음
   - [불만유형] 리스크 설명
     근거: 유사 리뷰 요약
     기획 보완: 구체적 수정 방향

   🟡 중간
   - ...

   🟢 낮음
   - ...

   **종합 의견**
   ...
   ```

## 입력이 없을 경우
사용법 안내: `/analyze-game [기획안 내용을 여기에 붙여넣기]`
